# Technical Code Assessments & Practice

This directory contains a collection of Python-based technical screenings and coding challenges modeled after real-world assessments from companies like **IBM, Amazon, and JPMorgan**.

## Core Focus
Unlike standard coursework, these projects emphasize **production-ready logic**:
*   **Defensive Programming**: Using `try/except` and `KeyError` handling to ensure scripts never crash on "dirty" real-world data.
*   **Data Aggregation**: Building complex summaries (Averages, Totals, Frequency) using only Python standard libraries.
*   **Relational Joins**: Connecting disparate datasets using dictionary-based lookups.
*   **Efficiency**: Utilizing Sets and Dictionaries for O(1) membership testing and data mapping.

## Practice Roadmap

### 1. [Beginner Questions](./Beginner%20Questions)
Fundamental exercises focused on Python syntax, list manipulation, and basic dictionary logic. These serve as the building blocks for complex data handling.

### 2. [General Assessment Practice](./General%20Assessment%20Practice)
A collection of "screening-style" problems focused on string parsing, cleaning messy logs, and basic ETL (Extract, Transform, Load) patterns.

### 3. [Fleet Practice](./Fleet%20Practice)
Logistics-heavy scenarios involving fleet management and warehouse operations. Key skills: Calculating averages across multiple datasets and implementing chronological date-filtering.

### 4. [IBM Practice](./IBM%20Practice)
High-level infrastructure and cloud-resource optimization scenarios. These mimic IBM-style screenings by requiring multi-dataset joins and advanced multi-level sorting (e.g., descending numeric sorts with alphabetical tie-breakers).
